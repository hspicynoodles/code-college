    Promises in Javascript are used to handle asynchronous operations. They are like a promise that something will happen in the future. Promises can be in three states: pending, fulfilled, or rejected. When a promise is pending, it means that the operation is still happening. When a promise is fulfilled, it means that the operation has completed successfully. When a promise is rejected, it means that the operation has failed.

    Why do we need promises? Promises are used to handle things suck as API calls, loading images, getting data from a database or even downloading a file. When we use promises are whenever we are doing something that takes time to complete. If we didnt use promises, our code would stop allowing for the browser to possibly freeze or crash.
